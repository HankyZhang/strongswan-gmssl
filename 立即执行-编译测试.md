# 立即开始编译测试

## ⚡ 快速执行命令

### 第1步：构建并启动Docker容器

```powershell
# 在PowerShell中执行（当前目录: C:\Code\strongswan）

# 构建Docker镜像
docker-compose -f docker-compose-build.yml build

# 启动容器
docker-compose -f docker-compose-build.yml up -d strongswan-build

# 查看容器是否运行
docker ps | Select-String "strongswan"
```

### 第2步：进入容器并编译

```powershell
# 进入容器
docker exec -it strongswan-gmsm-build bash

# === 以下命令在Docker容器内执行 ===

# 检查环境
gmssl version

# 运行编译（自动化脚本）
bash docker-build.sh

# 如果一切顺利，应该看到：
# ✓ GMSM插件已生成: ./src/libstrongswan/plugins/gmsm/.libs/libstrongswan-gmsm.so
```

### 第3步：测试插件

```bash
# 在容器内继续执行

# 运行测试脚本
bash docker-test.sh

# 期望看到 SM3, SM4 相关输出
```

### 第4步：手动验证（可选）

```bash
# 查找插件文件
find /strongswan -name "libstrongswan-gmsm.so" -ls

# 检查插件依赖
ldd src/libstrongswan/plugins/gmsm/.libs/libstrongswan-gmsm.so

# 检查符号
nm -D src/libstrongswan/plugins/gmsm/.libs/libstrongswan-gmsm.so | grep gmsm
```

```powershell
# 查看详细日志
docker-compose -f docker-compose-build.yml build --no-cache

# 检查网络连接（GmSSL需要从GitHub下载）
```

### 问题2：编译失败

```bash
# 在容器内查看详细configure输出
./configure --enable-gmsm --help

# 手动运行configure
./autogen.sh
./configure --enable-gmsm
make 2>&1 | tee build.log
```

### 问题3：找不到GmSSL

```bash
# 验证GmSSL安装
ls -la /usr/local/include/gmssl/
ls -la /usr/local/lib/libgmssl.*
pkg-config --modversion gmssl || echo "pkg-config未配置"
```

## 📊 完整的PowerShell命令序列（复制粘贴）

```powershell
# === 在PowerShell中执行 ===
cd C:\Code\strongswan

# 1. 构建镜像（首次需要约10分钟）
Write-Host "=== 构建Docker镜像 ===" -ForegroundColor Yellow
docker-compose -f docker-compose-build.yml build

# 2. 启动容器
Write-Host "`n=== 启动容器 ===" -ForegroundColor Yellow

Start-Sleep -Seconds 2

# 4. 检查容器状态
Write-Host "`n=== 容器状态 ===" -ForegroundColor Yellow
docker ps | Select-String "strongswan"

# 5. 进入容器
Write-Host "`n=== 进入容器 ===" -ForegroundColor Yellow
Write-Host "现在执行以下命令:" -ForegroundColor Green
Write-Host "  docker exec -it strongswan-gmsm-build bash" -ForegroundColor Cyan
Write-Host "然后在容器内运行:" -ForegroundColor Green
Write-Host "  bash docker-build.sh" -ForegroundColor Cyan
```
