### 2. Enum 枚举值处理 (6 个文件)

#### 2.1 src/libstrongswan/crypto/key_exchange.c
**错误**: `SM2_256` 未处理 (-Werror=switch)  
**修复**: 添加 SM2_256 case 验证 64 字节公钥
```c
case SM2_256:
    valid = value.len == 64;
    break;
```

#### 2.2 src/libstrongswan/crypto/hashers/hasher.c
**错误**: `HASH_SM3`, `SIGN_BLISS_*` 未处理  
**修复**: 5 个函数中添加 SM3 和 BLISS 相关 case
- hasher_algorithm_from_prf(): HASH_SM3
- hasher_algorithm_from_integrity(): HASH_SM3
- hasher_algorithm_to_integrity(): HASH_SM3
- hasher_algorithm_for_ikev2(): HASH_SM3
- hasher_from_signature_scheme(): SIGN_BLISS_*

#### 2.3 src/libstrongswan/crypto/iv/iv_gen.c
**错误**: `ENCR_SM4_CBC`, `ENCR_SM4_GCM_ICV16` 未处理  
**修复**: 添加 SM4 加密模式的 IV 生成
```c
case ENCR_SM4_CBC:
    return iv_gen_rand_create();
case ENCR_SM4_GCM_ICV16:
    return iv_gen_seq_create();
```

#### 2.4 src/libstrongswan/crypto/xofs/xof.c
**错误**: `HASH_SM3`, `HASH_MD2` 未处理  
**修复**: 添加对应的哈希算法支持
```c
case HASH_SM3:
    return hasher_create(HASH_SM3);
case HASH_MD2:
    return NULL;
```

#### 2.5 src/libstrongswan/credentials/keys/public_key.c
**错误**: `SIGN_SM2_WITH_SM3`, `SIGN_BLISS_*` 未处理  
**修复**: 添加签名方案支持
```c
case SIGN_SM2_WITH_SM3:
    return SIGN_SM2_WITH_SM3;
case SIGN_BLISS_WITH_SHA2_256:
case SIGN_BLISS_WITH_SHA2_384:
case SIGN_BLISS_WITH_SHA2_512:
    return sig;
```

### 3. 类型系统重构 (2 个文件)

#### 3.1 src/libstrongswan/plugins/gmsm/gmsm_sm2_dh.h
**错误**: strongSwan 6.x 重命名 `diffie_hellman_t` → `key_exchange_t`  
**修复**: 
- 添加 `#include <crypto/key_exchange.h>`
- 更新结构体定义和函数签名
- `diffie_hellman_group_t` → `key_exchange_method_t`

#### 3.2 src/libstrongswan/plugins/gmsm/gmsm_sm2_dh.c
**错误**: 接口方法名称变更  
**修复**: 更新所有 METHOD 宏和接口实现
- `set_other_public_value` → `set_public_key`
- `get_my_public_value` → `get_public_key`
- `get_dh_group` → `get_method`
- 删除 `set_private_value` (不再需要)

### 4. GmSSL 3.x HMAC API 适配 (2 个文件)

#### 4.1 src/libstrongswan/plugins/gmsm/gmsm_sm3_signer.c
**问题**: GmSSL 3.x HMAC_CTX 变为不透明类型  
**修复**:
1. 添加 `chunk_t key` 字段存储密钥
2. 更新 `hmac_init()` 调用：
   ```c
   // 旧: hmac_init(&ctx, key, len)
   // 新: hmac_init(&ctx, DIGEST_sm3(), key, len)
   ```
3. 更新 `hmac_finish()` 调用：
   ```c
   // 旧: hmac_finish(&ctx, mac)
   // 新: size_t mac_len = 32; hmac_finish(&ctx, mac, &mac_len)
   ```
4. 添加 `#include <gmssl/digest.h>` 获取 DIGEST_sm3()

#### 4.2 src/libstrongswan/plugins/gmsm/gmsm_sm3_prf.c
**问题**: 同上 HMAC API 变化  
**修复**: 相同的模式更新

### 5. GmSSL 3.x SM2 ECDH API 适配

#### 5.1 src/libstrongswan/plugins/gmsm/gmsm_sm2_dh.c
**问题**: `sm2_ecdh()` 函数签名改变  
**修复**:
```c
// 旧 API (GmSSL 2.x):
// sm2_ecdh(SM2_KEY *key, SM2_POINT *peer, uint8_t *out)

// 新 API (GmSSL 3.x):
// sm2_ecdh(const SM2_KEY *key, const uint8_t *peer_pub, size_t len, SM2_POINT *out)

// 实现:
if (sm2_ecdh(&this->private_key, value.ptr, value.len, &peer_point) != 1) {
    return FALSE;
}
memcpy(shared_key, &peer_point.x, 32); // 提取 X 坐标作为共享密钥
```

### 6. 宏重复定义

#### 6.1 src/libstrongswan/plugins/gmsm/gmsm_sm4_crypter.c
**错误**: `SM4_BLOCK_SIZE` 和 `SM4_KEY_SIZE` 重复定义  
**原因**: GmSSL 的 `<gmssl/sm4.h>` 已经定义这些常量  
**修复**: 删除本地定义，添加注释说明
```c
/* SM4_BLOCK_SIZE and SM4_KEY_SIZE are already defined in gmssl/sm4.h */
```

## 🔧 Docker 构建优化

### 多阶段构建策略
采用 4 阶段构建，优化缓存利用率：

1. **base**: Ubuntu 22.04 基础镜像
2. **dependencies**: 编译工具和依赖库（长期缓存）
3. **gmssl-builder**: 编译安装 GmSSL 3.1.1（版本缓存）
4. **strongswan-builder**: 编译安装 strongSwan（CACHE_BUST 控制）

### 缓存控制脚本
- **build-gmssl.ps1** (PowerShell): Windows 构建脚本
- **build-gmssl.sh** (Bash): Linux/macOS 构建脚本
- **参数**:
  - `-ForceUpdate`: 仅重新构建 strongSwan
  - `-NoCache`: 完全重新构建

### 构建时间对比
- 完全构建: ~8-10 分钟
- 仅更新代码: ~2-3 分钟（利用缓存）

## 🔍 技术要点总结

### 关键 API 变化

1. **strongSwan 5.x → 6.x**:
   - `diffie_hellman_t` → `key_exchange_t`
   - `diffie_hellman_group_t` → `key_exchange_method_t`
   - 接口方法重命名

2. **GmSSL 2.x → 3.x**:
   - HMAC_CTX 不透明化
   - `hmac_init()` 需要 digest 参数
   - `hmac_finish()` 使用输出长度指针
   - `sm2_ecdh()` 接受字节数组而非结构体

### 编译器严格性
- `-Werror=switch`: 所有 enum 必须显式处理
- `-Werror=implicit-function-declaration`: 必须声明函数
- `-Werror=incompatible-pointer-types`: 类型必须匹配
- `-Werror=int-conversion`: 禁止整数指针转换
